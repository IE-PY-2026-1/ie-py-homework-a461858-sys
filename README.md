# 🐍 [2026-01] 나만의 파이썬 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
* 오늘의 기분을 입력하면 행운의 색을 알려주는 투두리스트
<br>


## 2. 시나리오 (5~10줄)
* 이 프로그램은 사용자가 일상에서 해야 할 일들을 효율적으로 관리할 수 있도록 돕는 콘솔 기반 투두 리스트 시스템이다.
사용자는 할 일을 입력하고, 각 할 일에 대해 중요도를 설정할 수 있다. 입력된 할 일은 리스트에 저장되며, 언제든지 전체 목록을 확인할 수 있다. 각 할 일은 완료 여부를 체크할 수 있어 진행 상황을 한눈에 파악할 수 있다. 또한 중요도에 따라 할 일을 구분하여 조회할 수 있으며, 필요 없는 할 일은 삭제할 수 있다. 프로그램은 초기에는 단순한 입력과 출력 기능으로 시작하지만, 점차 리스트와 조건문을 활용하여 데이터를 관리하게 된다. 이후 반복문을 이용한 메뉴 기반 시스템으로 확장되며, 최종적으로는 함수와 자료구조를 활용하여 구조화된 프로그램으로 발전한다.

<br>


## 3. 예상 기능 및 메뉴 (최소 5개)
(예시)
1. 리스트 생성
2. 할 일 추가하기
3. 할 일 내용 입력
4. 오늘의 기분 입력
5. 행운의 색 확인
6. 할 일 완료 처리하기
7. 할 일 삭제하기
8. 프로그램 종료

## 🟦 [1차 과제: V1.0] 시나리오 기획
    
### **🤖 AI 파트너십 과정**
(예시)
 1. **내용 1 : 길드 매니저 컨셉 도출**
    * **프롬프트 요약:** "나는 대학생이고 콘솔 기반 투두리스트를 만들고 싶어. 어떤 방향으로 만들면 좋을까?"
    * **적용 내용:** 단순한 할 일 관리 프로그램이 아니라, 사용자 경험을 고려한 구조로 설계하는 방향을 제안받음. 기본적인 투두리스트 기능에 더해 카테고리 분류 및 부가 기능을 추가하는 아이디어를 도출함. 이 과정에서 input(), print()를 활용한 데이터 입출력과 변수 저장을 기본 구조로 설정함.
      
 2. **내용 2 : 로드맵 도출**
    * **프롬프트 요약:** "리스트를 여러 개 만들고 싶어. 공부, 할일, 오늘 기분 이런 식으로 나누고 싶어."
    * **적용 내용:** 카테고리별로 리스트를 분리하는 구조를 제안받아, 각 항목을 독립적으로 관리할 수 있도록 설계함. 이를 통해 프로그램의 가독성과 확장성을 높임. 이때 list 자료형을 활용하고, append(), remove() 등의 리스트 메서드와 if-elif-else 조건문을 이용해 카테고리 선택 기능을 구현하도록 계획함.
   
 3. **내용 3 : 재미 요소 및 기능 확장**
    * **프롬프트 요약:** "오늘의 행운의 색이나 조언 같은 것을 넣어도 좋지 않을까?"
    * **적용 내용:** ‘오늘의 행운의 색’과 ‘간단한 조언’을 리스트로 미리 만들어 두고, 그중 하나를 선택하여 출력하는 기능을 추가하도록 설계함. 이를 통해 프로그램에 재미 요소를 더하고 사용자 흥미를 높일 수 있도록 함. 구현 과정에서는 list 자료형과 input(), print()를 활용하고, if-elif-else 조건문과 while 반복문을 이용해 메뉴 선택에 따라 해당 기능이 실행되도록 구성함.

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
  * 이번 과제는 콘솔 기반 투두리스트 관리 시스템 구현으로, 사용자로부터 할 일과 관련 정보를 입력받아 리스트에 저장하고 이를 효율적으로 관리하는 프로그램을 구축하는 것입니다. 특히 if-elif 조건문을 활용하여 중요도에 따라 할 일을 볼 수 있고, 추후 오늘의 기분을 입력하면 하루의 행운의 색이 나오는 요소를 리스트와 조건문, input()을 활용해 구현할 예정이다.
  * input()을 통해 할 일 내용과 중요도(높음/보통/낮음)를 입력받아 리스트에 저장.
  * 리스트(list)를 활용하여 여러 개의 할 일을 관리하고, 카테고리(공부 / 할일 / 기분)에 따라 데이터를 구분하여 저장.
  * if-elif-else를 사용하여 중요도에 따라 다른 메시지를 출력하는 조건문 로직 구현.
  * 추가적으로 리스트에 저장된 데이터를 활용하여 ‘오늘의 행운의 색’과 간단한 조언을 출력하는 기능을 구현할 예정으로, 프로그램에 재미 요소를 더하고 사용자 참여도를 높이는 방향으로 확장함.
    
### **🤖 AI 파트너십 과정**
  
1. **내용 1: 리스트와 조건문을 결합한 확장 로드맵 자문**
   * **프롬프트 요약:** "파이썬 기초 문법 단계(리스트 → 조건문 → 반복문 → 함수)에 맞춰, '행운의 투두리스트'가 어떻게 발전할 수 있을지 4단계 확장 로드맵을 제안해줘."
   * **적용 내용:** 단순한 텍스트 출력을 넘어, '기초 정보 입력 및 변수 저장, 리스트와 조건문을 활용한 할 일 분류 및 중요도 판별(2차) → 반복문을 이용한 메뉴 시스템과 기분 기반 행운의 색 매칭(3차) → 딕셔너리와 함수를 활용한 구조화된 데이터 관리(4차)'로 이어지는 논리적인 확장 로드맵을 확립함.

2. **내용 2: 효율적인 데이터 분류 및 필터링 로직 도출**
   * **프롬프트 요약:** "2차 과제 수준에서 리스트와 조건문만 사용해 할 일을 효과적으로 관리할 방법이 있을까? 중요도에 따라 다른 메시지를 보여주고 싶어."
   * **적용 내용:** 입력받은 중요도 점수를 int()로 형변환하여 수치화하고, if-elif-else를 통해 점수가 높은 항목에는 "🔥 긴급"과 같은 상태 메시지를 부여하는 필터링 로직을 설계함. 또한 카테고리(공부, 운동 등)에 따라 맞춤형 한줄평을 출력하는 기능을 추가하여 조건문의 활용도를 높임.
    
### **🛠️ Troubleshooting & 기술 회고:**
1. **문제 1: 중첩 리스트의 특정 데이터에 접근할 때 발생하는 혼란 (IndexError/Logical Error)**
   * **원인:** 할 일을 [task, category, priority] 형태로 리스트 안에 리스트로 저장했는데, 중요도만 꺼내서 비교하고 싶을 때 인덱스 번호를 어떻게 써야 할지(todo_list[0][2] 등) 헷갈려 데이터가 잘못 출력되거나 오류가 발생함.
   * **해결:** 리스트의 구조를 시각화하여 첫 번째 대괄호는 '몇 번째 할 일'인지, 두 번째 대괄호는 '그 안의 구체적인 정보'인지를 파악함. 인덱스는 0부터 시작한다는 원칙을 다시 복습하여 정확한 위치의 데이터를 참조하도록 수정함.
2. **문제 2: 상세 조회(서브 메뉴) 시 입력값의 타입 불일치**
   * **원인:** 첫 번째 메뉴 선택(choice)은 문자열('2')로 처리하고, 이후 중요도를 선택하는 창에서 받는 값은 숫자(int)로 처리하는 과정에서 타입을 혼동하여 조건문이 항상 거짓이 되는 문제가 발생함.
   * **해결:** 비교 대상이 리스트 내부의 숫자 데이터라면 입력값도 반드시 int()로 형변환해야 한다는 점을 명확히 인지하고, 서브 메뉴의 입력값도 형변환을 적용하여 타입을 일치시켜 해결함.
